## 一、项目概况

完成电厂监盘智能化系统后端服务全套开发，基于 Spring Boot 3.2.5 + Java 17，共 **106 个 Java 源文件、54 个 API 接口、22 张数据库表**，编译通过、单元测试全部通过。

## 三、核心功能模块

### 1. 设备管理（4 接口）
- 机组 / 专业 / 系统 / 测点下拉列表
- Redis 缓存 30 分钟自动过期

### 2. 报警配置（10 接口）
- 49 条规则 CRUD，含多行规则子表一对多
- 分页+筛选（机组/专业/系统/类型/关键字）
- 批量删除、启用/禁用
- Excel 导入导出
- MapStruct DTO ↔ Entity ↔ VO 自动转换

### 3. 实时报警 + 分级（4 接口）
- 分页联表查询（关联维度表取名称）
- 报警确认（防重复 + 记操作人）
- 报警抑制（更新状态 + 写抑制记录）
- 三级分组卡片（一级/二级/智能预警，含 Top5）

### 4. 报警统计（7 接口）
- 概览（总数/未处理/抑制数/平均处理时长）
- 专业/级别/类型占比饼图
- 频次 Top10 / 时长 Top10
- 日趋势柱状图（按天+等级分组）
- Redis 缓存 5 分钟

### 5. 工况曲线 + 健康评分（2 接口）
- 曲线数据查询（支持 1h/6h/24h/7d 时间窗口）
- 设备健康评分（最新 + 历史趋势）

### 6. 历史数据导入（2 接口）
- ZIP/Excel 异步导入（@Async）
- 任务状态跟踪（等待→处理中→成功/失败）

### 7. 操作指导（2 接口）
- 本地优先查询，无则转发 KG 服务并落库
- 运行意见填写

### 8. Agent 会话（3 接口）
- 会话列表 / 消息列表 / 创建会话
- 预留 Agent 服务 HTTP 转发客户端

### 9. 内置 AI 引擎（5 接口）
- 报警智能诊断（严重程度评估 + 处置建议生成）
- 设备状态预测（健康分退化模型 + 7 天预测 + 风险等级）
- 根因分析（多证据因果推理 + 置信度评分）
- 通用分析入口（analysisType 切换）
- 历史分析结果追溯

### 10. 数据分析算法（5 接口）
- 异常检测（3-sigma 统计法，自动标记异常点）
- 趋势预测（最小二乘线性回归，R² 拟合优度）
- 健康评分计算（5 维加权算法：稳定性/频率/偏差/检修/环境）
- 报警关联分析（时间窗口共现关联）
- 实时数据统计（均值/最大/最小/标准差/变化率）

## 四、数据库设计

- **22 张表**：4 维度表 + 6 报警表 + 3 业务表 + 5 协作表 + 1 时序表 + 3 AI 结果表
- **双数据库适配**：MySQL（开发）+ PostgreSQL（生产），Flyway 双份迁移脚本
- **时序表**：`ts_measure_point`（PG 下为 TimescaleDB hypertable，7 天分区）
- **种子数据**：五大专业 + 2 台默认机组自动初始化
- **公共字段**：所有表含 `id / created_at / updated_at / deleted`（逻辑删除）

## 五、工程质量

- ✅ 统一响应体 `R<T>{ code, msg, data, timestamp }`
- ✅ `@RestControllerAdvice` 全局异常处理（业务/参数/数据库/系统异常）
- ✅ `@Validated` 参数校验 + 编码唯一性校验
- ✅ DTO / VO / Entity 三层分离，MapStruct 自动转换
- ✅ 事务保护（`@Transactional`），级联逻辑删除
- ✅ INFO 日志（操作人 + 目标 ID）
- ✅ 时区统一 Asia/Shanghai
- ✅ CORS 跨域配置
